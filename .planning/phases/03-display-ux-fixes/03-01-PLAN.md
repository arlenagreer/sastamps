<task type="auto">
  <name>Task 2: Normalize search scores to relative percentages capped at 100%</name>
  <files>js/modules/search-engine.js, js/pages/search.js</files>
  <action>
Search scores from Lunr.js are raw relevance scores that can exceed 1.0, causing display values like "2025%". Fix both display locations to use relative normalization where the highest score in the result set becomes 100% and all others are proportional.

**In js/pages/search.js** (displaySearchResults function, around line 248-303):

Before the results.map() call, compute the maximum score from the results array:
  const maxScore = Math.max(...results.map(r => r.score || 0), 1);

Then in the template literal where score is displayed (line ~284), change the score calculation from:
  Math.round(result.score * 100)
To:
  Math.round((result.score / maxScore) * 100)

Also change the label from "Relevance:" to just the percentage followed by "match" (e.g., "85% match").

**In js/modules/search-engine.js** (displaySearchResults and renderSearchResult methods):

The renderSearchResult method (around line 718) renders individual results but does not have access to the full result set for normalization. The normalization must happen in displaySearchResults (line ~686) which has access to all results.

1. In the displaySearchResults method, before calling renderSearchResult, compute the max score and pass it as a second argument.

2. Update renderSearchResult method signature to accept a maxScore parameter with default value of 1.

3. Change the score display calculation in renderSearchResult from:
   (result.score * 100).toFixed(1)
   To:
   Math.round((result.score / maxScore) * 100)

4. Change the label from "Relevance:" to just the percentage followed by "match".

This ensures the top result always shows "100% match" and others show proportional percentages. All user-facing text in the existing code already uses escapeHTML() for XSS prevention -- the score display is a computed number, not user input.
  </action>
