<task type="auto">
  <name>Task 1: Create the archived-newsletter.schema.json</name>
  <files>data/schemas/archived-newsletter.schema.json</files>
  <action>
Create `data/schemas/archived-newsletter.schema.json` following the conventions of existing schemas (Draft-07, additionalProperties: false).

The schema must validate:

1. **Top-level structure**: Object with required properties `archivedNewsletters` (array) and `metadata` (object)

2. **Entry schema** (items in archivedNewsletters array):
   - `id` (string, required): Pattern `^[0-9]{4}-(0[1-6]|Q[2-4]|special-covid)$`
   - `title` (string, required): Newsletter title
   - `year` (integer, required): minimum 2008, maximum 2024
   - `edition` (string, required): One of "01"-"06", "Q2", "Q3", "Q4", or "special-covid"
   - `editionLabel` (string, required): Human-readable label like "January/February" or "Second Quarter: April, May, June"
   - `filePath` (string or null, required): Pattern `^public/newsletter_archive/.*\.pdf$` when not null
   - `status` (string, required): enum ["available", "unavailable"]
   - `additionalProperties: false`

3. **Metadata schema**:
   - `lastUpdated` (string, format: date-time)
   - `totalIssues` (integer)
   - `availableIssues` (integer)
   - `yearRange` (object with `start` and `end` integers)
   - `version` (string)
   - `additionalProperties: false`

Follow the exact style of `data/schemas/newsletter.schema.json` for formatting and conventions.
  </action>
