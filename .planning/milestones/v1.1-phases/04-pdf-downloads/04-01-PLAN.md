<task type="auto">
  <name>Task 1: Create the download script with URL manifest parsing</name>
  <files>scripts/download-newsletters.mjs</files>
  <action>
Create `scripts/download-newsletters.mjs` -- a Node.js ES module script that:

1. **Extracts source HTML from git**: Run `git show HEAD:js/newsletter_archive/archived_newsletter_links.html` to get the download manifest. Parse the HTML to extract all `<a href="...">` links organized by year.

2. **Builds a download manifest array** with entries like:
```js
{ year, edition, label, sourceUrl, targetFilename }
```

The HTML structure is: nested `<li>` for years, inner `<li><a href="URL">Label</a></li>` for editions. Parse by tracking the current year context.

3. **Naming convention** (from CONTEXT.md decisions):
   - **Bimonthly (2008-2023):** `{year}-{nn}-{mon1}-{mon2}.pdf`
     - `nn` = two-digit edition number within the year (01-06)
     - Month abbreviations: jan, feb, mar, apr, may, jun, jul, aug, sep, oct, nov, dec
     - Map labels: "January/February" -> 01-jan-feb, "March/April" -> 02-mar-apr, etc.
   - **2008 special case:** Archive starts at edition 02 (Mar/Apr). Map: Mar/Apr=02, May/Jun=03, Jul/Aug=04, Sep/Oct=05, Nov/Dec=06
   - **2020 special edition:** "Special Covid-19 Edition" -> `2020-special-covid.pdf`. Insert between Mar/Apr (02) and May/Jun (03). After the special, continue numbering: May/Jun=03, Jul/Aug=04, Sep/Oct=05, Nov/Dec=06
   - **Quarterly (2024):** `{year}-q{n}-{mon1}-{mon2}-{mon3}.pdf`
     - "Second Quarter: April, May, June" -> `2024-q2-apr-may-jun.pdf`
     - "Third Quarter: July, Auguest, September" -> `2024-q3-jul-aug-sep.pdf` (note: source has typo "Auguest")
     - "Fourth Quarter: October, November December" -> `2024-q4-oct-nov-dec.pdf`

4. **URL conversion**:
   - Dropbox links: Replace `?dl=0` with `?dl=1` at end of URL
   - Google Drive links with format `drive.google.com/file/d/{FILE_ID}/view...`: Extract FILE_ID, use `https://drive.google.com/uc?export=download&id={FILE_ID}`
   - Google Drive links with format `drive.google.com/open?id={FILE_ID}`: Extract FILE_ID, use `https://drive.google.com/uc?export=download&id={FILE_ID}`

5. **Skip rules**:
   - Skip entries where href is `#` (2012 Jul/Aug "not available")
   - Skip all 2025 entries entirely (DL-05: already on site in public/)

6. **Download logic**:
   - Create `public/newsletter_archive/` directory if it doesn't exist
   - Use native `fetch()` (Node.js 18+) for downloads
   - Follow redirects (Dropbox and Google Drive both redirect)
   - Implement retry logic: 3 attempts with 2-second delay between retries
   - Download with limited concurrency (3 simultaneous downloads) to avoid rate limiting
   - After each download, validate the file starts with `%PDF` magic bytes (read first 4 bytes)
   - If validation fails, delete the corrupt file and log a warning
   - Log progress: `[X/N] Downloading year-edition.pdf... OK` or `FAILED`

7. **Summary output**: After all downloads complete, print:
   - Total downloaded successfully
   - Total skipped (with reasons)
   - Total failed (with filenames)

Import only Node.js built-in modules: `fs`, `path`, `child_process` (for git show). Use `node:` prefix for imports.
