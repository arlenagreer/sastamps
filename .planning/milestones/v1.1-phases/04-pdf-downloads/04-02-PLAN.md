<task type="auto">
  <name>Task 2: Validate archive completeness and naming</name>
  <files>public/newsletter_archive/*.pdf</files>
  <action>
Perform comprehensive validation of the downloaded archive:

1. **Count by year** -- verify expected editions per year:
   - 2008: 5 files (editions 02-06, no Jan/Feb)
   - 2009-2011: 6 files each
   - 2012: 5 files (missing Jul/Aug per DL-04)
   - 2013-2019: 6 files each
   - 2020: 7 files (6 regular + 1 special-covid)
   - 2021-2023: 6 files each
   - 2024: 3 files (Q2, Q3, Q4)
   - **Expected total: ~88 files**

2. **Naming validation** -- check all filenames match expected patterns:
   ```bash
   ls public/newsletter_archive/ | grep -E '^\d{4}-(0[1-6]-[a-z]{3}-[a-z]{3}|q[2-4]-[a-z]{3}-[a-z]{3}-[a-z]{3}|special-covid)\.pdf$'
   ```
   All files should match. Any non-matching filenames indicate naming bugs.

3. **Absence checks**:
   - `ls public/newsletter_archive/2012-04-jul-aug.pdf 2>/dev/null` -- should NOT exist (DL-04)
   - `ls public/newsletter_archive/2025-*.pdf 2>/dev/null` -- should NOT exist (DL-05)
   - `ls public/newsletter_archive/2008-01-jan-feb.pdf 2>/dev/null` -- should NOT exist (archive starts at Mar/Apr)

4. **PDF validity** -- spot-check a few files:
   ```bash
   for f in $(ls public/newsletter_archive/*.pdf | head -5); do
     xxd -l 4 "$f" | grep -q "2550 4446" && echo "$f: valid PDF" || echo "$f: INVALID"
   done
   ```

5. **No duplicates** -- verify no 2025 PDFs leaked into archive:
   ```bash
   ls public/newsletter_archive/2025*.pdf 2>/dev/null && echo "ERROR: 2025 files found" || echo "OK: no 2025 files"
   ```
