list pagination, breadcrumbs, and search filter controls -- with DaisyUI v5
equivalents styled to the Philatelic theme, including stamp-styled timeline
connectors and themed form controls.

### Task 15.2: Resource Accordion Swap (COMP-06)

**Files:** `resources.html`, `js/pages/resources.js`
**Effort:** Medium

#### DaisyUI Accordion Pattern

Replace the flat `.category-grid` of `.category-card` elements with a DaisyUI
`collapse` accordion grouped with `join join-vertical`.

```html
<div class="join join-vertical w-full stamp-accordion" id="resource-categories">
  <!-- One collapse per category, generated by JS -->
  <div class="collapse collapse-arrow join-item border-base-300 border bg-base-100">
    <input type="checkbox" name="resource-accordion" checked="checked" />
    <div class="collapse-title font-semibold">
      <i class="fas fa-book mr-2"></i> Getting Started
      <span class="badge badge-sm badge-primary ml-2">5 resources</span>
    </div>
    <div class="collapse-content">
      <!-- Individual resource-card items rendered here -->
      <div class="resource-card" data-id="..." data-category="getting-started"
           data-difficulty="beginner">
        <!-- resource content -->
      </div>
    </div>
  </div>
  <div class="collapse collapse-arrow join-item border-base-300 border bg-base-100">
    <input type="checkbox" name="resource-accordion" />
    <div class="collapse-title font-semibold">
      <i class="fas fa-star mr-2"></i> Grading & Condition
      <span class="badge badge-sm badge-primary ml-2">3 resources</span>
    </div>
    <div class="collapse-content">
      <!-- resources -->
    </div>
  </div>
  <!-- ... more categories -->
</div>
```

#### Migration Steps

1. In `resources.html`, replace the `<div class="category-grid">` container
   with `<div class="join join-vertical w-full stamp-accordion" id="resource-categories"></div>`.
2. In `js/pages/resources.js`, update `displayCategorizedResources()` to emit
   DaisyUI `collapse` markup per category:
   - Outer wrapper: `collapse collapse-arrow join-item border-base-300 border bg-base-100`
   - Hidden checkbox input: `<input type="checkbox" name="resource-accordion" />`
     (first category gets `checked="checked"`)
   - Title: `collapse-title font-semibold` with icon, category name, and badge count
   - Content: `collapse-content` containing individual resource cards
3. Use **checkbox** (not radio) inputs so multiple categories can be open simultaneously.
4. Update `initializeResourceFilters()`:
   - After applying filters, iterate each `.collapse` wrapper
   - Count visible `.resource-card` items in each `.collapse-content`
   - If zero visible, add `hidden` class to the collapse wrapper
   - If any visible, remove `hidden` class
5. Remove legacy `.category-grid`, `.category-card` CSS from `resources.html`
   inline styles.

#### Important Constraints

- **Resource card IDs and data attributes stay identical** -- `data-id`,
  `data-category`, `data-difficulty` selectors used by filter logic.
- **Bookmark functionality stays intact** -- bookmark buttons inside resource
  cards remain unchanged.
- **Modal integration stays intact** -- clicking resource details still calls
  `modal.open(data)`.

#### Acceptance Criteria

- Resource categories expand and collapse independently (checkbox behavior).
- Resource count badge shows correct number per category.
- Filtering by search text, category, and difficulty correctly shows/hides
  resources and auto-hides empty categories.
- All resource card interactions (bookmark, modal) continue to work.
